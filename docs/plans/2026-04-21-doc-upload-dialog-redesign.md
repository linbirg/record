# 证件上传弹窗交互优化设计

## 需求

优化车辆列表页「证件弹窗」的上传流程，使其更符合用户操作习惯。

## 当前问题

**现有流程：**
1. 选择证件名称（下拉框，默认"关系证明"，非第一个选项）
2. 点击「选择文件」按钮
3. 选择文件后立即上传（auto-upload="true"）

**问题：**
- 选择完图片就上传，用户没有机会修改证件名称
- 默认值不是第一个选项，不够直观
- 流程不符合"先选图片，再命名"的自然习惯

## 优化方案

### 调整后的流程

```
1. 点击「选择文件」按钮 → 打开文件选择器
2. 选择图片文件后 → 显示待上传区域（文件名 + 证件名称下拉框）
3. 确认证件名称（可修改下拉框，默认"驾驶证"）
4. 点击「确定上传」→ 执行上传
```

### 默认值调整

| 字段 | 当前值 | 优化后 |
|------|--------|--------|
| 证件名称 | "关系证明" | "驾驶证"（下拉框第一个选项） |

### UI 改动

1. **el-upload 配置**
   - `:auto-upload="false"` 改为手动上传
   - 移除自动上传触发

2. **新增上传确认区域**
   - 文件选择后，显示：文件名 + 证件名称下拉框 + 确定/取消按钮
   - 只有点击「确定上传」才执行上传

3. **上传完成后重置**
   - `docName` 恢复为默认值"驾驶证"
   - 清空待上传状态

## 技术实现

### 前端改动

**File:** `frontEnd/src/components/Container/car/CarList.vue`

1. `data` 新增字段：
   - `pendingFile`: 待上传文件对象
   - `pendingFileName`: 待上传文件名

2. 修改 `el-upload`:
   - `auto-upload` 设为 `false`
   - 绑定 `on-change` 处理文件选择

3. 新增确认上传区域：
   - 显示文件名
   - 证件名称下拉框（默认"驾驶证"）
   - 「确定上传」和「取消」按钮

4. 上传成功后重置状态

docName: "驾驶证",  // 改为第一个选项
